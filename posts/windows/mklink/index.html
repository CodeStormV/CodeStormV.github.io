MKLINK [[/D] | [/H] | [/J]] Link Target

        /D      创建目录符号链接。默认为文件
                符号链接。
        /H      创建硬链接而非符号链接。
        /J      创建目录联接。
        Link    指定新的符号链接名称。
        Target  指定新链接引用的路径
                (相对或绝对)。

使用 /d

创建目录符号链接。默认情况下，此命令创建文件符号链接。

只能链接文件夹 link 不占用硬盘空间
link 位置改变，链接失效，返回原来位置链接恢复 link 被复制：原文件内容被复制
link 改名，链接保持
原文件位置改变，链接失效，返回原来位置链接恢复
原文件改名，链接失效，改回名字链接恢复

使用 /j

创建目录交叉点。

只能链接文件夹
link 不占用硬盘空间
link 位置改变：link 移动到的位置变为文件夹而不是链接，将原文件内容移动到 link 移动后的文件夹内，但是原文件和原 link 保持不变，只是内容被转移
link 被复制：原文件内容被复制
link 改名，链接保持
原文件位置改变：链接失效，返回原来位置链接恢复
原文件改名：链接失效，改回原名链接恢复

使用 /h

创建硬链接而不是符号链接。

硬链接 只能链接文件
link 似乎占用空间
link 位置改变，链接不失效
link 被复制：原文件内容被复制
link 改名，链接保持 原文件改变位置，链接不失效
原文件改名，链接不失效

不使用

创建符号链接

符号链接 link 不占用硬盘空间
link 位置改变，链接失效，返回原来位置链接恢复
link 被复制：原文件内容被复制
link 改名，链接保持 原文件位置改变，链接失效，返回原来位置链接恢复
原文件改名：链接失效，改回原名链接恢复

总结

/d 与 /j 区别在于 link 位置改变之后内容是否转移。 /d 与 不使用 区别在于内容是 文件 还是 文件夹
